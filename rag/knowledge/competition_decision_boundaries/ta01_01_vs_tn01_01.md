# Decision

Choose `TA01_01` when one source repeatedly attempts authentication against the same target service in a short window, especially with high `failed_login_count`, elevated `attempt_rate`, short inter-attempt intervals, a failure burst, credential-field evidence, or a success-after-failures hint. FTP, SSH, HTTP login, RDP, SMB, and similar remote-access services can all carry brute-force evidence.

Choose `TN01_01` for a few normal logins, isolated retries, a single failed login, SSO redirects, health checks, or ordinary business access without sustained failure evidence. Reset-heavy traffic without authentication context is not brute force.

# Trigger cues

Retrieve this boundary for `auth_attempt_group`, `failed_login_count`, `attempt_rate`, `inter_attempt_intervals`, `failure_burst`, `success_after_failures_hint`, repeated FTP `530` responses, SSH auth failure hints, HTTP login paths, username/password field presence, or multiple same-source attempts to one authentication service.
