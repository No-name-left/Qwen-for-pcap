# Decision

Choose `TA43_01` for port or host discovery: many destination ports, many targets, short connections, high failed-connection ratio, SYN/UDP probe fanout, high probe rate, and bursty timing with little or no application payload. A broad `dst_port` fanout or target fanout stays `TA43_01` unless the record shows service-specific vulnerability probing.

Choose `TA43_02` only with vulnerability-discovery evidence: multiple URI paths on the same service, directory scan paths, high HTTP 404 probe pattern, scanner User-Agent, Nikto/OpenVAS/Nessus/Nmap NSE/sqlmap clues, CVE-like paths, plugin probes, version/banner fingerprinting, or repeated service-specific checks. A service name, one admin path, or port fanout alone is insufficient.

# Trigger cues

Retrieve this boundary when the record contains `same_src_unique_dst_ports`, `unique_dst_ports`, `same_src_unique_dst_ips`, `failed_conn_rate`, `probe_rate`, `burstiness_score`, URI fanout, `high_404_rate`, scanner User-Agent, CVE-like path, or probe path keywords such as phpMyAdmin, wp-admin, .git, CGI, HNAP, server-status, or directory traversal probes.
